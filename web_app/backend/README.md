# Infant Monitor — API Documentation

A Flask REST API for the hospital infant monitoring system. Receives telemetry from a Raspberry Pi, stores it in a MySQL database, and serves a dashboard for nurses.

## Authentication

Protected frontend routes require a **JWT Bearer token** in the request header.

### How to Get a Token

Call `POST /login` with valid credentials. Store the returned token and attach it to every subsequent request.

### Request Header Format

```
Authorization: Bearer <token>
```

### Token Details

- Expires after **30 minutes** (8 Hours for personal development)
- Contains nurse `id` and `name` (readable by frontend via base64 decode)
- Invalidated on logout via blocklist

### JavaScript Helper

```javascript
async function fetchWithAuth(url, options = {}) {
    const token = localStorage.getItem("access_token");

    const response = await fetch(url, {
        ...options,
        headers: {
            "Authorization": `Bearer ${token}`,
            "Content-Type":  "application/json",
            ...options.headers
        }
    });

    if (response.status === 401) {
        localStorage.removeItem("access_token");
        window.location.href = "/login";
        return null;
    }

    return response.json();
}
```

## HMAC Security

Pi routes are protected by **HMAC-SHA256** signature verification. Every request from the Pi must include a `signature` field in the JSON body.

### How to Sign a Payload

The signature is generated by:
1. Sorting the JSON keys alphabetically
2. Serialising to a compact string with no spaces
3. Computing HMAC-SHA256 using the shared `HMAC_SECRET`

### Python (Raspberry Pi side)

```python
import hmac, hashlib, json, os

SECRET_KEY = os.getenv("HMAC_SECRET")

def sign_payload(data: dict) -> str:
    body      = json.dumps(data, separators=(",", ":"), sort_keys=True)
    signature = hmac.new(SECRET_KEY.encode(), body.encode(), hashlib.sha256).hexdigest()
    return signature

# Usage
payload = {"rfidTagID": "ABCD1234", "timestamp": "2024-01-01T00:00:00Z"}
payload["signature"] = sign_payload(payload)
# send payload to API
```

### Node-RED (JavaScript function node)

```javascript
const crypto    = require("crypto");
const SECRET    = env.get("HMAC_SECRET");

const data      = { ...msg.payload };
const sorted    = Object.keys(data).sort().reduce((obj, key) => {
    obj[key] = data[key];
    return obj;
}, {});

const body      = JSON.stringify(sorted);
const signature = crypto.createHmac("sha256", SECRET).update(body).digest("hex");

msg.payload     = { ...data, signature };
msg.headers     = { "Content-Type": "application/json" };
return msg;
```

> **Note:** If `HMAC_SECRET` is not set on the server, signature verification is skipped. Always set it in production.

## Frontend Routes

Base URL: `/`

All routes except `/login` (POST) require JWT authorization.

---

### POST /login

Authenticate a nurse and receive a JWT token.

**Authorization required:** No

**Request body:**
```json
{
    "username": "user1",
    "password": "abcd1234"
}
```

**Success response `200`:**
```json
{
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "id": 1,
    "name": "Nurse Sarah"
}
```

**Error response `200`:**
```json
{
    "error": "Invalid credentials"
}
```

### GET /status

Returns the latest telemetry reading and whether the Pi is currently online (last reading within 60 seconds).

**Authorization required:** Yes

**Success response `200`:**
```json
{
    "id": 1,
    "presence": "infant_present",
    "state": "sleeping",
    "crying": false,

    "humidity": 60.2,
    "light": 430,
    "loudness": 320,
    "timestamp": "2024-01-01T00:00:00",

}
```

**Empty response `200`:** Returns empty string if no telemetry recorded yet.

### POST /checkin/update

Updates the action note on a check-in record.

**Authorization required:** Yes

**Request body (form data):**

| Field | Type | Description |
|---|---|---|
| `id` | integer | ID of the check-in record to update |
| `action` | string | Action note — alphanumeric, spaces, hyphens, underscores only to prevent XSS|

**Success response `200`:**
```json
{
    "status": "success"
}
```

**Validation error `401`:**
```json
{
    "status": "failed",
    "error": "Invalid input"
}
```

---

**Authorization required:** Yes

**Success response `200`:**

## Pi Routes

Base URL: `/api`

All Pi routes require a valid **HMAC signature** in the request body. No JWT required.

---

### POST /api/checkin

Records a nurse check-in when an RFID tag is tapped. Looks up the nurse by RFID tag ID and creates a check-in record.

**Authorization required:** HMAC signature

**Request body:**
```json
{
    "rfidTagID": "ABCD1234",
    "timestamp": "2024-01-01T14:30:00Z",
    "signature": "a3f7b2c9d1e4..."
}
```

| Field | Type | Description |
|---|---|---|
| `rfidTagID` | string | RFID tag ID of the nurse |
| `timestamp` | string | ISO 8601 timestamp from Pi |
| `signature` | string | HMAC-SHA256 signature of payload |

**Success response `200`:**
```json
{
    "status": "success"
}
```

**Unauthorized response `401`:**
```json
{
    "error": "Unauthorized"
}
```

> If the RFID tag is not found in the database the request still returns success — no record is created but no error is raised.

---

| `infantState` | string | Detected state at time of alert |


**Unauthorized response `401`:**

**Success response `200`:**
```json
{
```

**Unauthorized response `401`:**

## Error Responses

### JWT Errors (Frontend Routes)

| Error | Status | Response |
|---|---|---|
| No token provided | `401` | `{"error": "No token provided"}` |
| Invalid token | `401` | `{"error": "Invalid token"}` |
| Token expired | `401` | `{"error": "Token has expired"}` |
| Token revoked (logout) | `401` | `{"error": "Token has been revoked"}` |

### HMAC Errors (Pi Routes)

| Error | Status | Response |
|---|---|---|
| Missing or invalid signature | `401` | `{"error": "Unauthorized"}` |

### Validation Errors

| Error | Status | Response |
|---|---|---|
| Invalid action input | `401` | `{"status": "failed", "error": "Invalid input"}` |
